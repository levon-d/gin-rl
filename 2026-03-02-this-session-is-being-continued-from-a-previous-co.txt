⏺ High-Level Overview: GIN-RL Experiment System

  The Goal

  Your dissertation is about using Reinforcement Learning (RL) to intelligently
  select which mutation operators to use when trying to improve Java programs.
  The goal is to reduce memory usage of target methods.

  ---
  What is Genetic Improvement (GI)?

  Genetic Improvement is a technique that automatically modifies source code to
  improve some property (speed, memory, energy). It works like this:

  1. Take a working program
  2. Make small random changes (mutations) to the code
  3. Test if the mutated code still works (passes tests)
  4. Measure if it's better (uses less memory)
  5. Keep improvements, discard failures
  6. Repeat

  ---
  Where Does TestRunListener Come In?

  TestRunListener is a JUnit test listener. It hooks into JUnit's test
  execution:

  // When a test STARTS running:
  public void executionStarted(TestIdentifier testIdentifier) {
      startAllocatedBytes = threadMXBean.getThreadAllocatedBytes(currentThread);
  }

  // When a test FINISHES running:
  public void executionFinished(TestIdentifier testIdentifier, ...) {
      endAllocatedBytes = threadMXBean.getThreadAllocatedBytes(currentThread);
      memoryUsed = (endAllocatedBytes - startAllocatedBytes) / 1024;  // Convert
   to KB
  }

  It's not "just a test" - it's the instrumentation that captures metrics while
  tests run. GIN uses JUnit tests as a way to:
  1. Verify the code still works correctly after mutation
  2. Exercise the code so we can measure its resource usage

  ---
  How Memory is Measured

  Old approach (broken):
  // Heap snapshot - measures total JVM heap state
  Runtime.getRuntime().totalMemory() - Runtime.getRuntime().freeMemory()
  // Problem: For fast tests, this barely changes (0-1 MB)

  New approach (working):
  // Thread allocation tracking - measures bytes actually allocated
  ThreadMXBean.getThreadAllocatedBytes(threadId)
  // Tracks every object allocation made by this specific thread
  // Much more accurate: 220 KB → 7 KB measurable

  ---
  What UCB (Upper Confidence Bound) Does

  UCB balances exploration (trying new operators) vs exploitation (using known
  good operators):

  UCB(operator) = AverageReward(operator) + c * sqrt(ln(total_steps) /
  times_used)
                   └─────────────────────┘
  └────────────────────────────────────┘
                     Exploitation term              Exploration bonus
                     (higher if operator           (higher if operator
                      has worked well)              hasn't been tried much)

  The algorithm picks the operator with the highest UCB score. Over time, it
  learns to favor operators that produce better improvements while still
  occasionally trying less-used operators.

  1. Goal: Reduce memory usage of JCodec methods using Genetic Improvement
  2. Innovation: Use RL (UCB) to intelligently select mutation operators
  3. TestRunListener: Captures memory usage during test execution
